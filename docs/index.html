Everything You Need

A complete backend platform for real-time multiplayer games — from state synchronization to player matchmaking.

Real-Time State Sync

Sub-millisecond state synchronization across all connected nodes using CRDT-inspired conflict resolution and delta compression.

Game Server Orchestration

Create, manage, and scale game servers dynamically. Supports CS2, FiveM, and any custom game engine via REST or WebSocket.

Smart Matchmaking

Skill-based matchmaking with configurable queues, rating thresholds, and automatic match creation when enough players queue up.

Persistence & WAL

Write-Ahead Log and automatic state snapshots ensure zero data loss. Full event sourcing support for state replay and debugging.

Security & Auth

JWT authentication, role-based access control, bcrypt password hashing, and configurable rate limiting out of the box.

Monitoring & Alerts

Prometheus-compatible metrics, configurable alerting, health checks, and a beautiful real-time web dashboard for full observability.

WebSocket Protocol

Binary-optimized WebSocket transport with connection pooling, automatic reconnection, and message batching for maximum throughput.

Cluster Coordination

Distributed node management with leader election, health monitoring, and automatic failover for high-availability deployments.

REST Admin API

Full-featured REST API for server management, entity CRUD, matchmaking, metrics, alerts — all secured with JWT authentication.

Built for Scale

A modular, event-driven architecture designed for distributed real-time workloads.

REST Server Management

bash 
# Authenticate
curl -X POST localhost:8080/api/v1/auth/login \
  -d '{"username":"admin","password":"admin123"}'

# Create a game server
curl -X POST localhost:8080/api/v1/servers \
  -H "Authorization: Bearer TOKEN" \
  -d '{"mode":"ranked","map":"dust2","maxPlayers":10}'

# List all servers
curl localhost:8080/api/v1/servers \
  -H "Authorization: Bearer TOKEN"

# Queue player for matchmaking
curl -X POST localhost:8080/api/v1/matchmaking/enqueue \
  -H "Authorization: Bearer TOKEN" \
  -d '{"playerId":"p1","mode":"ranked","skillRating":1500}'

WS Real-Time Sync

javascript 
const ws = new WebSocket('ws://localhost:8080');

// Authenticate
ws.send(JSON.stringify({
  type: 'auth',
  token: 'YOUR_JWT_TOKEN'
}));

// Create a player entity
ws.send(JSON.stringify({
  type: 'entity_create',
  entityType: 'player',
  data: { name: 'Alice', hp: 100 }
}));

// Update entity
ws.send(JSON.stringify({
  type: 'entity_update',
  entityId: 'ENTITY_ID',
  operations: [
    { op: 'set', path: 'hp', value: 75 }
  ]
}));

API Endpoints

Method Endpoint Description
POST /auth/login Authenticate & get JWT token
GET /servers List all game servers
POST /servers Create a game server
GET /state/entities List all entities (filterable)
POST /matchmaking/enqueue Queue player for matchmaking
GET /dashboard/overview Full system overview
GET /metrics Prometheus-compatible metrics

Up and Running in Minutes

Three commands and you have a full real-time game backend with a dashboard.

1

Install & Build

git clone https://github.com/alexltz0/SYNC_ENGINE.git
cd sync-engine
npm install
npm run build
2

Start the Engine

npm start
# Server running on http://localhost:8080
# WebSocket on ws://localhost:8080
3

Open the Dashboard

cd dashboard
npm install && npm run dev
# Dashboard on http://localhost:3001
# Login: admin / admin123

🐳 Or use Docker 

docker-compose up -d
# That's it. Backend + Dashboard running.
